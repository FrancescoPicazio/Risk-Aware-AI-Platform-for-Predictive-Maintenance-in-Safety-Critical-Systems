# Contributing
Thank you for your interest in contributing! 
## How to Contribute
1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Follow the architectural principles in [ARCHITECTURE.md](docs/ARCHITECTURE.md)
4. Commit with clear messages: `git commit -m "feature: description"`
5. Push to your fork and submit a Pull Request
## Guidelines
- Follow PEP 8 style guidelines
- Include docstrings for functions and classes
- Add type hints where applicable
- Update documentation as needed
- Ensure tests pass: `pytest tests/`
## Reporting Issues
- Use the bug report template on GitHub
- Include steps to reproduce and environment details
- Check existing issues before creating duplicates
## Code of Conduct
Be respectful and constructive in all interactions.
